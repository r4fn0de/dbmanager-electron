# Plano: Edição inline no painel de detalhes da linha (Row Details Overlay)

## Contexto

O `TableEditorRowDetailsOverlay` atualmente mostra campos da linha como texto estático. O `CellExpandPopover` já possui toda a lógica de edição por tipo (JSON, array, date, bool, UUID, etc.) com validação e inputs específicos. O objetivo é tornar o painel de detalhes editável, reutilizando essa infraestrutura.

## Escopo

1. Extrair utilitários do `CellExpandPopover` para reutilização
2. Criar componente `EditableField` para um campo único
3. Modificar `TableEditorRowDetailsOverlay` para suportar edição
4. Integrar com `TableDataEditor` passando callbacks necessários

---

## Passo 1: Extrair utilitários do CellExpandPopover

**Arquivo**: `src/features/database/components/CellExpandPopover.tsx`

Mover as seguintes funções para um novo arquivo `src/features/database/components/table-editor-utils.ts`:

- `valueToEditableText()`
- `initialToUtcIso()`, `utcIsoToDatetimeLocal()`, `datetimeLocalToUtcIso()`
- `timestampRawToDatetimeLocal()`, `datetimeLocalToTimestamp()`
- `initialDate()`, `initialTime()`, `initialBool()`, `initialNumeric()`
- `ValidationResult` type e `validateDraft()`
- Constantes: `NULL_SENTINEL`, `UUID_RE`, `IPV4_RE`, `IPV6_RE`, `MACADDR_RE`, `INTERVAL_TEXT_RE`, `INT_RANGE`
- `intRangeFor()`, `isValidIp()`, `isValidPgArrayLiteral()`

Importar essas funções de volta no `CellExpandPopover` para não quebrá-lo.

**Por quê**: O painel de detalhes precisa da mesma classificação de tipo e validação que o popover. Sem extração, duplicaríamos ~500 linhas.

## Passo 2: Criar componente EditableField

**Arquivo novo**: `src/features/database/components/TableDataEditor/components/EditableField.tsx`

Props:
```ts
interface EditableFieldProps {
  column: SchemaColumn;
  value: unknown;
  readOnly?: boolean;
  onSave: (rawText: string) => void; // "NULL" para nulo
}
```

Comportamento:
- Usa `classifyColumnKind()` para determinar o tipo de input
- Usa `valueToEditableText()` para inicializar
- Usa `validateDraft()` para validar em tempo real
- Renderiza o input apropriado (igual ao `CellExpandPopover`):
  - `json`/`array`: textarea com Monaco JSON/plaintext
  - `timestamptz`/`timestamp`/`date`/`time`: input datetime/date/time
  - `bool`: botões TRUE/FALSE
  - `integer`/`numeric`: input number
  - `uuid`: input text + botão Generate
  - `bytea`/`inet`/`cidr`/`macaddr`/`interval`/`enum`: input text com placeholder
  - `text` (default): textarea
- Estado local: `draft`, `isNullDraft`, `validation`
- Botões: "Set NULL", "Cancel", "Save"
- Auto-focus e select-all ao iniciar edição
- `Esc` cancela, `Cmd+Enter` salva

**Nota**: Este componente é uma versão inline/simplificada do `CellExpandPopover` — sem o wrapper Popover, sem header de coluna, focado apenas no input e ações.

1. **Nova prop para o overlay**:
   ```ts
   const handleFieldSaveInOverlay = useCallback((columnName: string, rawText: string) => {
     if (!expandedRow) return;
     if (expandedRow.rowKey.startsWith("insert:")) {
       const insertIndex = Number(expandedRow.rowKey.slice(7));
       applyExpandedEditToInsert(insertIndex, columnName, rawText);
     } else {
       const entry = effectiveRows.find((r) => r.rowKey === expandedRow.rowKey);
       if (entry) {
         applyExpandedEditToRow(expandedRow.rowKey, entry.row, columnName, rawText);
       }
     }
   }, [expandedRow, effectiveRows, applyExpandedEditToRow, applyExpandedEditToInsert]);
   ```

2. **Passar `columns` para o overlay**:
   - `columns={table.columns}`
   - `readOnly={primaryKey.length === 0}`
   - `onFieldSave={handleFieldSaveInOverlay}`

## Trade-offs considerados

| Decisão | Alternativa | Motivo |
|---------|-------------|--------|
| Extrair utils para arquivo separado | Duplicar lógica no overlay | Manter DRY, evitar drift |
| Criar `EditableField` componente | Inline no overlay | Reutilizável, testável, isolado |
| Auto-save em blur | Botão explícito por campo | Pattern do CellExpandPopover, explícito e seguro |
| "Save All" no footer do painel | Salvar campo a campo | Consistente com o footer do grid (TableEditorFooter) |
| Manter div estática em read-only | Sempre editável | Sem PK não pode salvar, read-only evita confusão |

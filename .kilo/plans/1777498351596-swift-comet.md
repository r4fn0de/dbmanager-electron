# Plan: Persistent Custom AI Models

## Problem
- Models are hardcoded in `PROVIDERS` in `src/ipc/ai/config.ts` ŌĆö no way to add custom models
- OpenAI-compatible fetched models are stored in React state (`availableModels`) ŌĆö lost on reload
- Users cannot add custom model IDs to any provider

## Solution

### 1. Add `customModels` to persisted settings (`src/ipc/ai/config.ts`)

- Add `customModels: Record<string, string[]>` to `AiSettings` interface (maps provider name ŌåÆ list of custom model IDs)
- Default: `{}`
- Add helper functions:
  - `addCustomModel(provider, modelId)` ŌĆö appends to the array (dedup)
  - `removeCustomModel(provider, modelId)` ŌĆö removes from the array
- Update `isModelAllowedForProvider()` to also check `customModels[provider]`
- Update `getProvidersInfo()` to merge custom models into each provider's model list (mark them with `isCustom: true`)

### 2. Add IPC handlers (`src/ipc/ai/handlers.ts` + `src/ipc/ai/index.ts`)

- `aiAddCustomModel` ŌĆö input: `{ provider, modelId }` ŌåÆ calls `addCustomModel()`
- `aiRemoveCustomModel` ŌĆö input: `{ provider, modelId }` ŌåÆ calls `removeCustomModel()`
- Register both in the `ai` router object

### 3. Add frontend IPC proxies (`src/features/ai/hooks/ai-actions.ts`)

- `addCustomModel(provider, modelId)` ŌĆö calls `ipc.client.ai.addCustomModel()`
- `removeCustomModel(provider, modelId)` ŌĆö calls `ipc.client.ai.removeCustomModel()`

### 4. Update `AiSettingsPanel` (`src/features/ai/components/AiSettingsPanel.tsx`)

**In the provider config dialog, model section:**
- Show the merged model list (built-in + custom). Custom models get a small "├Ś" remove button
- Add an input field + "Add" button below the model list to add a custom model ID
- For **OpenAI-compatible**:
  - "Fetch models" button now **persists** fetched models as custom models via `addCustomModel()`
  - Fetched models appear as regular models in the list and survive restarts
  - Remove the `availableModels` local state ŌĆö use `settings.providers[].models` (which now includes custom models)
- Custom models are visually distinguishable (e.g., subtle badge or dimmer text)

### 5. Files changed

| File | Change |
|------|--------|
| `src/ipc/ai/config.ts` | Add `customModels` field, add/remove helpers, update `isModelAllowedForProvider`, update `getProvidersInfo` |
| `src/ipc/ai/handlers.ts` | Add `aiAddCustomModel`, `aiRemoveCustomModel` handlers |
| `src/ipc/ai/index.ts` | Register new handlers in router |
| `src/features/ai/hooks/ai-actions.ts` | Add `addCustomModel`, `removeCustomModel` proxy functions |
| `src/features/ai/components/AiSettingsPanel.tsx` | Custom model add/remove UI, persist fetched models |

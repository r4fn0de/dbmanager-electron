# Plano de Enriquecimento — TarsDB AI & Features

## Sumário

Propostas divididas em 3 categorias: **AI enrichment** (melhorias no assistente existente), **AI-powered features** (novas funcionalidades com AI em partes do app que ainda não usam), e **Standalone features** (funcionalidades úteis sem AI).

---

## PARTE 1 — AI Enrichment (melhorias no AI já existente)

### 1.1 Fix SQL com schema context
**Problema**: `fixSql()` no passa `schemaContext` para o modelo. A AI corrige SQL "cega", sem saber quais tabelas/colunas existem.
**Solução**: Passar `schemaContext` no payload de `fixSql` (igual `updateSql` j faz). Ajustar handler em `src/ipc/ai/handlers.ts` e `ai-actions.ts`.
**Impacto**: Correes muito mais precisas. **Esforo**: Pequeno.

### 1.2 Slash Commands no chat
**Problema**: No h atalhos rpidos para tarefas comuns no chat.
**Soluo**: Implementar comandos `/<command>` no prompt input:
- `/explain <sql>` — explica SQL em portugus ingls
- `/optimize <sql>` — sugere otimizao com EXPLAIN
- `/schema-review` — analisa schema atual e sugere melhorias
- `/generate-test-data <table>` — gera INSERT com dados sintticos
- `/doc <table>` — gera documentao da tabela
- `/migrate <instruction>` — gera script de migrao (ALTER TABLE etc)
- `/profile <table>` — profiling de dados (nulos, padres, outliers)
**Impacto**: UX muito mais fluida, o user no precisa escrever prompts longos. **Esforo**: Mdio.

### 1.3 Follow-up Suggestions (sugestes ps-resposta)
**Problema**: Aps cada resposta do AI, o usurio precisa pensar no que perguntar.
**Soluo**: Aps cada resposta, gerar 2-3 sugestes de follow-up usando `generateText` (temperature alta). Exemplos:
  - "Show me the indexes on this table"
  - "Suggest a query to find duplicates"
  - "Generate a migration to add a column"
Renderizar como botes clicveis abaixo da resposta.
**Impacto**: Engajamento, descoberta de features. **Esforo**: Mdio.

### 1.4 Saved Prompts / Prompt Templates
**Problema**: Usurios repetem os mesmos prompts (ex: "otimizar query", "gerar dados de teste").
**Soluo**: Templates salvos pelo usurio + templates built-in. Renderizar como chips no empty state do chat ou via dropdown no input. Persistir em localStorage.
**Impacto**: Produtividade. **Esforo**: Pequeno.

### 1.5 AI Explain Query (no SQL editor)
**Problema**: No h boto "Explain this SQL" com AI diretamente na toolbar do SQL editor. O user precisa ir ao chat.
**Soluo**: Boto na toolbar do SqlEditor (prximo ao "Fix SQL") que envia o SQL selecionado + EXPLAIN plan para o AI e renderiza a explicao em popover ou inline.
**Impacto**: Workflow contnuo sem sair do editor. **Esforo**: Mdio.

### 1.6 AI Optimize Query (no SQL editor)
**Problema**: Sem sugesto de otimizao rpida no editor.
**Soluo**: Boto na toolbar que roda EXPLAIN + pede ao AI para analisar o plano e sugerir melhorias (indexes, reescrita). Renderiza sugestes como code actions ou painel lateral.
**Impacto**: Performance tuning acessvel. **Esforo**: Mdio.

### 1.7 Inline AI em SQL existente (seleo)
**Problema**: Inline AI s funciona com editor vazio. No d para selecionar parte do SQL e pedir "adicione WHERE" ou "converta para CTE".
**Soluo**: Permitir que o inline prompt aparea quando h texto selecionado. O `sql` enviado ao backend  o texto selecionado, e o resultado substitui apenas a seleo (no editor todo).
**Impacto**: Edio incremental de SQL com AI. **Esforo**: Pequeno.

### 1.8 Conversation export/sharing
**Problema**: No d para exportar uma conversa.
**Soluo**: Boto de exportar conversa para Markdown (j existe download parcial em `conversation.tsx`). Expandir para incluir tool calls, resultados SQL, e formatao completa.
**Impacto**: Documentao, compartilhamento. **Esforo**: Pequeno.

## PARTE 2 — AI-Powered Features (AI em partes do app sem AI)

### 2.1 AI Filters no Table Data Editor
**Problema**: `getAiFilters` j existe em `ai-actions.ts` mas no  usado em nenhum lugar. O TableDataEditor tem apenas filtro "contains" bsico.
**Soluo**: Integrar `getAiFilters` no TableDataEditor. Boto "AI Filter" que abre prompt, converte linguagem natural em filtros estruturados (column, operator, value). Substitui/aplica no filtro existente.
**Impacto**: Alta visibilidade, funcionalidade j 80% pronta. **Esforo**: Pequeno.

### 2.2 AI Schema Review / Health Check
**Problema**: No h anlise automtica de sade do schema.
**Soluo**: Painel ou ao no DatabaseOverview que roda AI analysis do schema completo: missing indexes, naming conventions, normalization issues, unused columns, data type suggestions. Renderiza como lista de "findings" com severidade e sugestes.
**Impacto**: Diferencial competitivo forte. **Esforo**: Mdio.

### 2.3 AI Generate Test/Sample Data
**Problema**: Para testar, usurios precisam criar dados manualmente.
**Soluo**: No TablesExplorerSidebar, opo "Generate sample data" que usa AI para gerar INSERTs realistas baseados no schema da tabela (types, constraints, FK relationships). O AI gera dados respeitando FKs, CHECK constraints, enums.
**Impacto**: Muito til para devs em desenvolvimento. **Esforo**: Mdio.

### 2.4 AI-Assisted DDL (Create Table from description)
**Problema**: CreateTableDialog  puramente manual. No d para descrever a tabela em linguagem natural.
**Soluo**: No CreateTableDialog, adicionar campo "Describe your table" que usa AI para gerar colunas iniciais baseado na descrio. O usurio pode refinar manualmente depois.
**Impacto**: Acelera criao de tabelas. **Esforo**: Mdio.

### 2.5 AI Schema Documentation Generator
**Probleo**: Tabelas no tm documentao. No h como gerar docs do schema.
**Soluo**: Gerar documentao automaticamente: descrio de cada tabela, colunas com propsito, relacionamentos. Exportar como Markdown, HTML, ou dbdocs (DBML). Integrar no SchemaExportDialog.
**Impacto**: Documentao de DB  sempre negligenciada; isso resolve o problema. **Esforo**: Mdio.

### 2.6 AI no Schema Visualizer
**Problema**: Schema Visualizer  puramente visual, sem AI.
**Soluo**: Permitir perguntas sobre o diagrama: "Which tables have no indexes?", "Show me orphan tables", "Suggest relationships". Implementar como chat overlay ou painel lateral no visualizer com contexto do diagrama atual.
**Impacto**: Mdio. **Esforo**: Mdio.

## PARTE 3 — Standalone Features (sem AI, teis para o usurio)

### 3.1 Schema Diff / Migration Generator
**Problema**: No h como comparar dois schemas ou gerar migrations.
**Soluo**: Ferramenta que compara schemas entre:
  - Duas connections (ex: staging vs production)
  - Duas branches do mesmo local DB
  - Schema atual vs snapshot salvo
Gera script SQL de migrao (ALTER TABLE, CREATE TABLE etc). UI com diff visual (added/removed/modified columns, indexes, constraints).
**Impacto**: Altssimo para devs que gerenciam migrations. **Esforo**: Grande.

### 3.2 Data Comparison Tool
**Problema**: No d para comparar dados entre tabelas.
**Soluo**: Comparar dados entre a mesma tabela em dois databases/schemas. Mostra diff visual (rows added, removed, modified). til para validar migrations.
**Impacto**: Alto para QA e validation. **Esforo**: Grande.

### 3.3 Visual Query Builder
**Problema**: SQL editor requer conhecimento de SQL.
**Soluo**: Builder visual drag-and-drop: selecionar tabelas, arrastar colunas, definir joins visualmente, filtros com dropdowns, agregaes. Gera SQL automaticamente. Integrado ao SqlEditor como modo alternativo.
**Impacto**: Democratiza o uso para no-SQL experts. **Esforo**: Grande.

### 3.4 Backup/Restore para Local DBs
**Problema**: Local DBs no tm backup. Perder dados  fcil.
**Soluo**: Boto de backup no DatabaseOverview para local databases: `pg_dump` (PostgreSQL) ou file copy (SQLite). Restore com validao. Agendamento opcional.
**Impacto**: Segurana de dados. **Esforo**: Mdio.

### 3.5 Query Performance Dashboard
**Problema**: No h monitoramento de performance.
**Soluo**: Painel no DatabaseOverview mostrando:
  - Slow queries (pg_stat_statements, performance_schema)
  - Active connections com query em execuo
  - Lock contention
  - Cache hit ratio (j existe parcialmente no overview)
  - Table bloat
**Impacto**: Visibilidade operacional. **Esforo**: Mdio.

### 3.6 Enhanced Import/Export
**Problema**: S h CSV import/export. No suporta JSON, SQL dumps, Excel.
**Soluo**: Wizard de import/export com mltiplos formatos:
  - JSON, JSONL
  - SQL dump (INSERT statements)
  - Excel (.xlsx)
  - Parquet (para data analytics)
  - Com mapeamento de colunas e preview
**Impacto**: Interoperabilidade. **Esforo**: Mdio.

### 3.7 Global Schema Search (cross-connection)
**Problema**: Busca de tabelas/colunas  por connection.
**Soluo**: Search global (Cmd+Shift+F ou Cmd+P) que busca em todas as connections ativas. til para encontrar onde uma tabela ou coluna especfica existe. Mostra resultados com connection label + schema + table.
**Impacto**: Produtividade em mltiplos databases. **Esforo**: Mdio.
